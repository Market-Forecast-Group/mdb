020 * <p>
021 * An MDB file does not contain any meta-data, it is just raw data, 
022 * one record next to the other, every sinlge byte is part of the data,
023 * however, this class contains the required information to "understand"
024 * the files format. Important, do not try to access files created 
025 * by other classes because you will get an unexpected behavior and corrupted data.
026 * </p>
027 * <p>
028 * This is the schema this "driver" class understands:
029 * </p>
030 * <h3>Demo_02_Price definition</h3>
031 * <table border=1>
032 *      <caption>Demo_02_Price</caption>
033 *      <tr>
034 *              <td>Column</td>
035 *              <td>Type</td>
036 *              <td>Order</td>
037 *              <td>Virtual</td>
038 *              <td>Formula</td>
039 *      </tr>
040 * <tr>
041 *              <td>time</td>
042 *              <td>INTEGER</td>
043 *              <td>ASCENDING</td>
044 *              <td>No</td>
045 *              <td></td>
046 *      </tr>
047 * <tr>
048 *              <td>price</td>
049 *              <td>LONG</td>
050 *              <td>NONE</td>
051 *              <td>No</td>
052 *              <td></td>
053 *      </tr>
054 * </table>
055 * <h3>Demo_02_PriceMDB API</h3>
056 * <p>
057 * Now let's see the operations you can perform using this class on Demo_02_Price files:
058 * </p>
059 *
060 * <h3>Data Insertion</h3>
061 *
062 * <p>
063 * As you know, in MDB you cannot insert a record in the middle of the file, all the data you add to the file is at the end of it, for that reason
064 * we named {@link Appender} to the component in charge of this function. So to add data to the file you have to request an appender:
065 * </p>
066 * <pre>
067 * // To connect to an MDB file, the best is to use
068 * // the "connect" method of the session.
069 * // In the next examples, we will asume you know it.
070 * 
071 * Demo_02_MDBSession session = ...;
072 * Demo_02_PriceMDB mdb = session.connectTo_Demo_02_PriceMDB("demo_02_price.mdb");
073 * 
074 * // request the appender.
075 * Demo_02_PriceMDB.Appender app = mdb.appender(); 
076 *
077 * // set the appender values
078 * app.time = ...;
079 * app.price = ...;
080
081 * // add the record
082 * app.append();
083 *
084 * ...
085 *
086 * // You can repeat this operation for each item you want to add to the file.
087 * // Important, is possible that some of these records are yet in the memory buffer
088 * // so to writes them to the file, you have to flush the appender:
089 *
090 * app.flush();
091 *
092 * ...
093 *
094 * // When you are sure you do not want to add new records, close the appender.
095 * // The close method also write the pending records to the disk, 
096 * // so it is not needed to call the flush method.
097 *
098 * app.close();
099 * </pre>
100 * <p>
101 * So use the method {@link #appender()} to get the appender, and use the 
102 * methods {@link Appender#append()}, {@link Appender#append(Record)} and 
103 * {@link Appender#append_ref_unsafe(Record)} to add the records.
104 * </p>
105 *
106 * <h3>Data Query</h3>
107 * <p>
108 * We provide different APIs to retrieve data: cursors, selection methods and List wrappers. 
109 * All of them are based on cursors, so it is important you understand how cursors work.
110 * </p>
111 * <h4>Cursors</h4>
112 * <p>
113 * There are two type of cursors:
114 * </p>
115 * <ul>
116 * <li>{@link Cursor}: sequential cursor.</li>
117 * <li>{@link RandomCursor}: random access cursor.</li>
118 * </ul>
119 * <p>
120 * Both cursors have a particular function and you must use the more appropiate 
121 * depending on the problem. 
122 * </p>
123 *
124 * <h5>Sequential Cursor</h5>
125 *
126 * <p>
127 * This is the common cursor, it retrieves all the records from a start position to an stop position.
128 * The API is very simple, in the following example we print the data from "start" to "stop":
129 * </p>
130 * <pre>
131 * Demo_02_PriceMDB mdb = ...;
132 * long start = ...;
133 * long stop = ...;
134 *
135 * // request a sequential cursor from start to stop
136 * Demo_02_Price.Cursor cursor = mdb.cursor(start, stop);
137 *
138 * // iterate the records from start to stop
139 * while (cursor.next()) {
140 *      // print the content of the current record
141 *      System.out.println("Read "  
142 *                      + cursor.time + " "
143 *                      + cursor.price + " "
144 *              );
145 * }
146 * // important always close the cursor
147 * cursor.close();
148 * </pre>
149 * <p>
150 * As you can see it is very simple, just to highlight that you should close the cursor
151 * when you stop using it. A cursor creates certain OS resources that should be released as
152 * soon the cursor is not needed anymore, also in certain operating systems like Windows, you 
153 * cannot delete the underlaying file until it gets released.
154 * <p> 
155 * See the {@link Cursor} javadoc for more details.
156 * </p>
157 * 
158 * <h5>Random Access Cursor</h5>
159 * <p>
160 * This cursor provides random access to the file. It is possible to implement a 
161 * random access method using sequential cursors (open it, read a record, and close), 
162 * but a {@link RandomCursor} is the API we provide
163 * to perform this task in a more efficient way.
164 * </p>
165 * <p>
166 * Let's see this API with an exmaple:
167 * </p>
168 * <pre>
169 * Demo_02_PriceMDB mdb = ...;
170 *
171 * // request a random cursor
172 * Demo_02_PriceMDB.RandomCursor cursor = mdb.randomCursor();
173 *
174 * // read record at position 10
175 * cursor.seek(10);
176 * System.out.println(cursor.toRecord());
177 *
178 * // read record at position 34
179 * cursor.seek(34);
180 * System.out.println(cursor.toRecord());
181 *
182 * // remember always to close the cursor
183 * cursor.close();
184 * </pre>
185 * <p>
186 * In our experience, many times you would like to keep alive a random access cursor
187 * until the session gets closed. In this case, we recommend to "defer" the cursor.
188 * A deferred cursor is not more than a cursor that is closed automatically
189 * when the session is closed.
190 * See the {@link MDBSession#defer(ICursor)} javadoc for more information.
191 * </p>
192 * <p>
193 * MDB uses random access cursors to implement other APIs with a higher level of abstraction
194 * like the {@link MDB#record(IRandomCursor, long)} method and the lists. Now we want to focus
195 * on the "record" method. It is very easy to use, the previous example can be re-implemented
196 * in this way:
197 * </p>
198 * <pre>
199 * Demo_02_PriceMDB mdb = ...;
200 * // read record at position 10
201 * System.out.println(mdb.record(10));
202 *
203 * // read record at position 34
204 * System.out.println(mdb.record(34));
205 * </pre>
206 * <p>
207 * Is it simpler right? And it performs very well, yet a random cursor is a bit faster
208 * but you can use the "record" if you have a deadline, just do not use to retrieve
209 * sequential, in that case remember to use a sequential cursor or one of the "select"
210 * methods (we explain later these methods).
211 * </p>
212 * <p>
213 * Something to highlight about the "record" method implementation, it creates 
214 * a random-deferred cursor per thread, so it i safe if many threads call it 
215 * at the same time.
216 * </p>

224 * <h3>Data Update</h3>
225 *
226 * <p>
227 * MDB provides two type of methods to update the values:
228 * </p>
229 * <ul>
230 *      <li>
231 *              Update a unqiue row. It re-writes the content of the whole record. 
232 *              It is available for tables without array definitions.
233 * </li>
234 *      <li>
235 *              Update a particular column or a unique row. It re-writes only that field of the record. 
236 *              It is available only for non-array columns.
237 *      </li>
238 * </ul>
239 * <p>
240 * You see in both cases it updates only one record at the same time, and the value to replace should be primitive.
241 * Also remember virtual columns are not updated because its values are computed automatically, they are not stored physically.
242 * </p>
243 * <p>
244 * The API is simple:
245 * </p>
246 * <pre>
247 * Demo_02_PriceMDB mdb = ...;
248 * // the index of the record you want to update/replace.
249 * long index = ...;
250 *
251 * // the new values                                                                            
252 * int new_val_time = ...;
253 * long new_val_price = ...;
254 *
255 * mdb.replace(index 
256 *              , new_val_time
257 *              , new_val_price
258 *              );
259 * </pre>
260 * <p>
261 * If you want to update just one column of the record, then you may use the following methods:
262 * </p>
263 * <ul>
264 * <li>{@link Demo_02_PriceMDB#replace_time(long, int)}: To replace the time value.</li>

266 * </ul>

560                /**
561                * Append a new record to the file with the appender's values. 
562                */
563                @Override
564                public void append() throws IOException {
565                        if (_basic) {
566                                try {
567                                        assert _rbufSize == 0 && _rbuf == null : "In basic mode the shared buffer is empty";
568                                
569                                        /* basic append, do not put the record in memory */
570                                        if (_buf.position() == _buf.capacity()) {
571                                                flush();
572                                        }
573
574                                
575                                        _buf.putInt(this.time);
576                                        _buf.putLong(this.price);
577
578                                        _size++;
579                                        
580                                        return;
581                                } catch (Exception e) {
582                                        _size = fsize();
583                                        throw e;
584                                }
585                        }
586
587                                /* regular append, put the record in the shared buffer */
588                                _writeLock.lock();
589                                try {
590                                        if (_rbufSize == _rbuf.length) {
591                                                if (_memory) {
592                                                        int newSize = _rbufSize * 2;
593                                                        Record[] b = new Record[newSize];
594                                                        System.arraycopy(_rbuf, 0, b, 0, _rbufSize);
595                                                        for(int i = 0; i < _rbufSize; i++) {
596                                                                b[_rbufSize + i] = new Record();
597                                                        }
598                                                        _rbuf = b;
599                                                } else {
600                                                        flush();
601                                                }
602                                        }       
603                                        Record r = _rbuf[_rbufSize];
604                                        r.time = this.time;
605                                        r.price = this.price;
606                                        _rbufSize++;
607                                        _size++;
608                                } catch (Exception e) {
609                                        _size = fsize() + _rbufSize;
610                                        throw e;
611                                } finally {
612                                        _writeLock.unlock();
613                                }
614                }

701                /**
702                * Write pending records, it clears the buffer.
703                * @throws IOException If there is any I/O error.
704                */
705                public void flush() throws IOException {        
706                        if (_basic) {
707                                _buf.limit(_buf.position());
708                                _buf.rewind();
709                                _channel.write(_buf);
710                                _buf.limit(_buf.capacity());
711                                _buf.rewind();
712                                _rbufPos = fsize();
713                                return;
714                        }
715                        
716                        if (_memory) {
717                                return;
718                        }
719
720                        for(int j = 0; j < _rbufSize; j++) {                                            
721                                Record r = _rbuf[j];
722                                _buf.putInt(r.time);
723                                _buf.putLong(r.price);
724                        }
725                        _buf.rewind();
726                        _buf.limit(_rbufSize * 12);
727                        _channel.write(_buf);
728                        _buf.limit(_buf.capacity());
729                        _buf.rewind();
730                        
731                        _writeLock.lock();              
732                        try {                   
733                                _rbufSize = 0;
734                                _rbufPos = fsize();
735                        } finally {
736                                _writeLock.unlock();
737                        }
738                }
